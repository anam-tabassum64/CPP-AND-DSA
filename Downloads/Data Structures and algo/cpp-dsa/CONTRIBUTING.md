# Contributing to cpp-dsa

Thanks for helping improve this repository.

## How to contribute

1. Open an issue for a missing topic, bug, or improvement idea.
1. Keep changes focused on one topic or one improvement at a time.
1. Use clear file names and consistent folder placement.
1. Prefer small, readable examples over large dumps of code.
1. Add a short explanation before adding code samples.

## Content rules

- Use C++17 style by default
- Keep examples beginner-friendly when possible
- Add complexity notes for algorithms and data structures
- Include edge cases and assumptions where useful
- Avoid copying large blocks from external sources without attribution

## Suggested workflow

1. Fork or branch from `main`
1. Create your changes
1. Verify formatting and correctness
1. Submit a pull request with a short summary

## Pull request checklist

- The file is in the correct folder
- Naming follows the repository convention
- The content is useful and concise
- Any code compiles or is clearly marked as pseudocode
- TODO items are left only for future work that is intentionally deferred
