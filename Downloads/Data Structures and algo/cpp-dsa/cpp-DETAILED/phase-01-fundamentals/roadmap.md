# Phase 1 - Learning Roadmap

This roadmap gives the correct learning order for Phase 1 of C++ Fundamentals.

The sequence is arranged so that each topic prepares you for the next one.

## Learning Sequence

| Step | Topic | Why It Comes Here |
| --- | --- | --- |
| 1 | Introduction to C++ | Start with the language purpose, scope, and use cases before writing code |
| 2 | History of C++ | Understanding evolution helps explain modern features and design choices |
| 3 | Features of C++ | Learn what makes C++ different from other languages |
| 4 | Applications of C++ | See where the language is used in industry and problem solving |
| 5 | Compilation Process | Learn how source code becomes an executable before writing programs |
| 6 | Source File | Understand what a `.cpp` file is and how it participates in compilation |
| 7 | Preprocessor | Learn preprocessing before diving into compiler behavior |
| 8 | Compiler | Understand translation of C++ code into machine-oriented output |
| 9 | Assembler | Learn how intermediate representation becomes assembly-level instructions |
| 10 | Linker | Understand how multiple files and libraries are combined |
| 11 | Loader | See how the executable is placed into memory for execution |
| 12 | Variables | Start with storage of data once the toolchain is understood |
| 13 | Data Types | Learn what kinds of data variables can hold |
| 14 | Literals | Understand fixed values before combining them into expressions |
| 15 | Constants | Learn how to protect values from accidental change |
| 16 | Keywords | Recognize reserved words before writing valid syntax |
| 17 | Identifiers | Learn naming rules for variables, functions, and namespaces |
| 18 | Namespaces | Reduce name conflicts after learning identifiers |
| 19 | Input & Output | Begin interacting with the user after learning basic syntax |
| 20 | Operators | Use values in calculations and comparisons |
| 21 | Expressions | Combine operators, values, and variables into meaningful logic |
| 22 | Type Conversion | Understand how C++ converts between types during expression evaluation |
| 23 | Scope | Learn where variables can be accessed after understanding declarations |
| 24 | Lifetime | Learn how long objects remain valid in memory |
| 25 | Storage Classes | Understand how storage duration and visibility differ |
| 26 | Control Flow | Move into decision making once the syntax base is ready |
| 27 | `if` | Start with the simplest decision statement |
| 28 | `switch` | Learn structured branching after basic conditions |
| 29 | Loops | Use repetition after learning branching |
| 30 | `break` | Learn early exit control inside loops and switches |
| 31 | `continue` | Learn how to skip iterations safely |
| 32 | `goto` | Study only for educational understanding of control flow |
| 33 | Functions | Introduce reusable logic after mastering control flow |
| 34 | Function Declaration | Separate interface from implementation |
| 35 | Function Definition | Write the actual function body after declaration |
| 36 | Function Prototype | Learn how compilers validate function usage early |
| 37 | Function Overloading | Extend function design after understanding function signatures |
| 38 | Inline Functions | Learn small performance-oriented function definitions |
| 39 | Default Arguments | Make function calls simpler and more flexible |
| 40 | Command Line Arguments | Learn how programs receive input before execution |
| 41 | Basic Debugging | Practice diagnosing issues after writing complete programs |
| 42 | Assertions | Add checks to verify program assumptions during development |

## Recommended Study Flow

1. Read the introduction, history, features, and applications sections together
2. Study the compilation process and understand each stage in order
3. Learn variables, data types, literals, constants, keywords, and identifiers
4. Study namespaces, input/output, operators, and expressions
5. Move into type conversion, scope, lifetime, and storage classes
6. Learn control flow with `if`, `switch`, loops, `break`, `continue`, and `goto`
7. Finish with functions, prototypes, overloading, inline functions, default arguments, command-line arguments, debugging, and assertions

## Milestones

- I can explain the C++ compilation pipeline
- I can write and run simple programs without help
- I can choose appropriate data types and variable names
- I can write conditions and loops correctly
- I can create and call reusable functions
- I can debug basic syntax and logic issues
- I am ready to move to arrays and strings
