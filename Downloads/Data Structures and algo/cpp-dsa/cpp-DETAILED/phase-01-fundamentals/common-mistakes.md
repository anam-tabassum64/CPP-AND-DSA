# Phase 1 - Common Mistakes

## Beginner Mistakes

- Forgetting semicolons
- Using undeclared variables
- Writing uninitialized variables
- Confusing assignment with equality
- Missing braces in control statements

## Runtime Mistakes

- Division by zero
- Accessing invalid memory later in the course
- Infinite loops
- Reading invalid input values

## Compilation Errors

- Missing headers
- Wrong function signatures
- Misspelled identifiers
- Using keywords as variable names
- Incorrect syntax in `if`, `switch`, or loops

## Logic Mistakes

- Wrong condition in `if`
- Off-by-one errors
- Incorrect loop bounds
- Forgetting to return from a function
- Misunderstanding operator precedence

## Debugging Strategies

- Read the error message carefully
- Reduce the problem to the smallest failing example
- Print intermediate values
- Use assertions for assumptions
- Test boundary cases

## Interview Mistakes

- Not explaining the compilation process clearly
- Confusing syntax knowledge with problem-solving skill
- Not knowing the difference between scope and lifetime
- Skipping basic examples during revision

## Best Practices

- Keep programs small while learning
- Use descriptive names
- Write one concept at a time
- Revisit failed examples until they feel natural
